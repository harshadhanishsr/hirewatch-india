⚡ Ingestion Layer — AWS

  • EventBridge rule triggers Lambda every hour (configurable)
  • Lambda #1: Scrapes Naukri job listings (company + role + location)
  • Lambda #2: Pulls LinkedIn public job data via structured HTTP
  • Lambda #3: Fetches EPFO ECR filing changes (ground truth headcount)
  • Lambda #4: Scrapes ET/Moneycontrol headlines for layoff signals
  • Lambda #5: MCA21 company master delta (new registrations)
  • All data lands raw to S3 with no transformation at this stage

⚙️ Databricks Processing

  • Auto Loader watches S3 bronze prefix, micro-batch trigger on new files
  • Bronze → Silver: clean, normalize, deduplicate, classify events
  • Entity resolution: fuzzy-match company names to MCA21 master
  • Location normalization: raw text → pincode → district → state
  • Event classification: HIRING / LAYOFF / NEUTRAL per record
  • Silver → Gold: district summaries, state rollups, company profiles
  • dbt models for reproducible, version-controlled transformations

🌐 Serving — Zero-ops API

  • Databricks job writes Gold Delta tables as static JSON to S3
  • Files: /api/india.json, /api/states/{id}.json, /api/districts/{id}.json
  • CloudFront distribution serves these files with 1-min TTL
  • Dashboard fetches from CloudFront — no DB, no server, near-zero cost
  • Optional: FastAPI on Lambda for dynamic queries (filters, search)
  • All endpoints public, no authentication required

🔄 Orchestration & Reliability

  • Databricks Workflows: visual DAG, retry policies, alerting
  • EventBridge + Lambda for ingestion scheduling (1hr default)
  • Dead-letter queues (SQS) for failed Lambda invocations
  • S3 raw layer = replay source; any stage can re-run from bronze
  • Pipeline is idempotent — re-running any batch is safe
  • Slack/email alerts on pipeline failures via Databricks notifications
  • Data freshness badge in dashboard: shows time since last batch
Intelligence Layer

Built-in AI & ML Models

Every Databricks processing run passes data through these models to enrich raw signals into actionable intelligence.

🏷️

Named Entity Recognition (NER)

Fine-tuned BERT model (HuggingFace) extracts company names, locations, and headcount figures from unstructured news headlines and job postings. Runs on Databricks GPU cluster.

Output → Structured company events with confidence scores
🔗

Entity Resolution

TF-IDF + fuzzy matching maps 50+ variants of the same company name ("TCS", "Tata Consultancy", "TCS Ltd") to a canonical company ID in the MCA21 master. Confidence threshold: 0.75+.

Output → Deduplicated company profiles with MCA21 linking
📉

Trend Forecasting (Prophet)

Facebook Prophet model trained on 12-month rolling window of hiring/layoff signals per state and sector. Predicts next 30-day trajectory. Updated every batch run with new actuals.

Output → 30-day forecasts per state, district, sector
😤

Sentiment Analysis

DistilBERT sentiment model classifies news articles about companies as positive/negative/neutral. Aggregated into a "company health score" that weights recent news more heavily.

Output → Health score (0–100) per tracked company
🚨

Anomaly Detection

Isolation Forest model flags statistical anomalies — sudden layoff spikes, unusual hiring surges, or silent companies going active. Triggers real-time alerts shown in the dashboard.

Output → Anomaly events with severity tags
📍

Location Intelligence

Custom NER + India Post pincode lookup converts raw location strings ("Whitefield", "BKC", "Sector 62") into verified district and state codes. Powers the geographic drill-down.

Output → Verified district_code, state_code per company record

Automation & Schedule

Pipeline Timing & Cost

Every component runs automatically. You never touch it once it's live.

⏱️

Ingestion — Every Hour

AWS EventBridge triggers all Lambda scrapers at the top of each hour. Each Lambda runs for ~2–4 minutes and dumps raw JSON to S3. Configurable to 30-min or 15-min intervals if needed (more Lambda cost).

EventBridge cron: 0 * * * *
⚙️

Databricks Workflow — Every Hour +5 min

Databricks job triggers 5 minutes after ingestion completes. Auto Loader detects new S3 files, runs Bronze → Silver → Gold pipeline. Total runtime: ~8–12 minutes including AI enrichment.

Databricks cron: 5 * * * *
🌐

Dashboard Refresh — On Data Write

The last step in the Databricks Workflow writes fresh JSON files to S3. CloudFront TTL of 5 minutes means public users see new data within 5 minutes of the batch completing. No manual refresh needed.

CloudFront TTL: 300s
🤖

AI Models — Every Batch Run

NER, sentiment, and entity resolution run on every batch. Prophet forecasting re-trains every 24 hours on the full history (more expensive, runs nightly). Anomaly detection runs every batch on the delta records only.

Forecast retrain: 0 2 * * *

Final Stack

Complete Tech Stack

Ingestion

  • AWS Lambda (Python 3.12)
  • AWS EventBridge Scheduler
  • BeautifulSoup4 + httpx (scraping)
  • AWS S3 (boto3)
  • AWS SQS (dead-letter queue)

Data Engineering

  • Databricks (Apache Spark 3.5)
  • Delta Lake (ACID transactions)
  • Databricks Auto Loader
  • dbt (data transformations)
  • Databricks Workflows (orchestration)

AI / ML

  • HuggingFace Transformers (NER, sentiment)
  • Prophet (time series forecasting)
  • scikit-learn (entity resolution, anomaly)
  • Databricks MLflow (model registry)
  • Databricks Feature Store

Serving

  • AWS S3 (Gold JSON files)
  • AWS CloudFront (edge CDN)
  • Optional: FastAPI on AWS Lambda
  • Cloudflare (DNS + SSL)

Frontend

  • Next.js 14 (React)
  • Leaflet.js (map)
  • Chart.js / Recharts (charts)
  • Tailwind CSS (styling)
  • Vercel (hosting, free tier)

DevOps / IaC
